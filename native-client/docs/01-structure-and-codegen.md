### Why these choices

| Package | Replaces | Rationale |
|---|---|---|
| `dio` | `fetch()` | Interceptors, typed responses, cancellation, works on all platforms |
| `web_socket_channel` | browser `WebSocket` | Platform-agnostic, supports binary, works on desktop + mobile |
| `flutter_riverpod` | React hooks | 1:1 mapping to existing hook pattern (see [Phase 4](./04-state.md)) |
| `freezed` + `json_serializable` | `types.gen.ts` | Immutable data classes with JSON serialization, generated from Pydantic |
| `record` | `getUserMedia` | Native mic capture on all 6 platforms, supports raw PCM output |
| `file_picker` | `<input type="file">` | Cross-platform file selection |
| `just_audio` | `AudioContext.decodeAudioData` | MP3 decode + audio playback |
| `mocktail` | vitest mocks | Dart-native mocking, no codegen needed |

## 1.3 Dart Codegen from Pydantic

### Current flow (TypeScript)

```
Pydantic models → codegen.py → client/src/api/types.gen.ts
```

### New flow (Dart)

```
Pydantic models → codegen.py --dart → native-client/lib/models/models.dart
                                  ↓
                            build_runner → models.freezed.dart + models.g.dart
```

### Extending `codegen.py`

Add a `generate_dart()` function (~100 lines) to the existing `server/src/codegen.py`. It walks the same `SHARED_MODELS` list and emits Freezed-annotated Dart classes.

**Pydantic model:**
```python
class SessionStatus(StrEnum):
    CREATED = "created"
    ACTIVE = "active"
    PAUSED = "paused"
    CLOSED = "closed"

class Session(BaseModel):
    id: str
    pipeline_id: str
    label: str = ""
    status: SessionStatus = SessionStatus.CREATED
    sample_rate: int = 48000
    channels: int = 1
    created_at: float
    stats: SessionStats
```

**Generated Dart:**
```dart
// AUTO-GENERATED by server/src/codegen.py — do not edit by hand

import 'package:freezed_annotation/freezed_annotation.dart';
import 'package:json_annotation/json_annotation.dart';

part 'models.freezed.dart';
part 'models.g.dart';

enum SessionStatus {
  @JsonValue('created') created,
  @JsonValue('active') active,
  @JsonValue('paused') paused,
  @JsonValue('closed') closed,
}

@freezed
class Session with _$Session {
  const factory Session({
    required String id,
    @JsonKey(name: 'pipeline_id') required String pipelineId,
    @Default('') String label,
    @Default(SessionStatus.created) SessionStatus status,
    @JsonKey(name: 'sample_rate') @Default(48000) int sampleRate,
    @Default(1) int channels,
    @JsonKey(name: 'created_at') required double createdAt,
    required SessionStats stats,
  }) = _Session;

  factory Session.fromJson(Map<String, dynamic> json) =>
      _$SessionFromJson(json);
}
```

### Key mapping rules

| Pydantic | Dart |
|---|---|
| `StrEnum` | `enum` with `@JsonValue` annotations |
| `BaseModel` | `@freezed` class |
| `snake_case` field | `camelCase` field + `@JsonKey(name: 'snake_case')` |
| `str` | `String` |
| `int` | `int` |
| `float` | `double` |
| `field = default` | `@Default(value)` |
| `field: T \| None = None` | `T?` (nullable, no `@Default`) |
| Nested `BaseModel` | Reference to other Freezed class |

### Commands

```bash
# Generate Dart models from Pydantic
cd server && uv run python -m src.codegen --dart

# Run build_runner to produce .freezed.dart and .g.dart
cd native-client && dart run build_runner build --delete-conflicting-outputs

# Check Dart models are not stale (for CI)
cd server && uv run python -m src.codegen --dart --check
```

### Output path

`codegen.py` writes to `native-client/lib/models/models.dart`. The `--dart` flag selects Dart output; the existing `--check` flag verifies staleness. Without `--dart`, the existing TypeScript codegen runs (backward compatible).
